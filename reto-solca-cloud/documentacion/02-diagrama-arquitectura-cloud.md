## Capas de la Arquitectura

### 1. Capa de Infraestructura (IaaS)
- **Docker Engine**: ContenedorizaciГіn de todos los servicios
- **VolГәmenes persistentes**: Almacenamiento de datos MySQL
- **Red virtual**: `solca-network` para comunicaciГіn interna entre contenedores
- **OrquestaciГіn**: Docker Compose para desarrollo, Docker Swarm/Kubernetes para producciГіn

### 2. Capa de Plataforma (PaaS)
- **Spring Boot 3.2**: Framework de desarrollo de microservicios
- **Actuator**: Monitoreo de salud de cada servicio
- **Flyway**: Migraciones de base de datos automГЎticas
- **Maven**: GestiГіn de dependencias y construcciГіn

### 3. Capa de Software (SaaS)
- **Microservicio Pacientes**: GestiГіn del maestro de pacientes
- **Microservicio Consultas**: Historial de consultas clГӯnicas
- **Microservicio Laboratorio**: Resultados de laboratorio
- **Microservicio ImagenologГӯa**: Estudios de imagen (PACS simulado)
- **Microservicio Repositorio**: AgregaciГіn y consulta consolidada

### 4. Capa de Base de Datos (DBaaS)
- **MySQL 8.0**: Motor de base de datos relacional
- **4 bases independientes**: Una por dominio de negocio
- **Flyway Migrations**: Control de versiones de esquemas

## Flujo de Datos

```
PeticiГіn HTTP вҶ’ API Gateway вҶ’ Microservicio especГӯfico вҶ’ MySQL propia
                              вҶ“
                    Repositorio ClГӯnico вҶ’ Consulta a todos los servicios
                                       вҶ’ ConsolidaciГіn de resultados
                                       вҶ’ Respuesta Гәnica al cliente
```

## Principios de DiseГұo

- **Desacoplamiento**: Cada microservicio tiene su propia base de datos
- **Tolerancia a fallos**: Si un servicio falla, el repositorio responde con datos parciales
- **Escalabilidad horizontal**: Cada servicio puede escalar independientemente
- **API First**: ComunicaciГіn exclusivamente por API REST
- **Seguridad por capas**: Red interna, autenticaciГіn por servicio, HTTPS
